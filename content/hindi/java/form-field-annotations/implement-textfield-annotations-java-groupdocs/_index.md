# Java PDF Form Annotations: इंटरैक्टिव PDFs बनाएं जो उपयोगकर्ता वास्तव में भरना चाहते हैं

## क्यों आपके PDFs को इंटरैक्टिव फ़ॉर्म फ़ील्ड्स की आवश्यकता है (और उन्हें कैसे जोड़ें)

क्या आपने कभी ऐसा PDF फ़ॉर्म भरने की कोशिश की है जो इंटरैक्टिव नहीं था? आप प्रक्रिया जानते हैं - डाउनलोड करना, प्रिंट करना, हाथ से भरना, स्कैन करना, और ईमेल से वापस भेजना। अब 2025 है, और आपके उपयोगकर्ता बेहतर की उम्मीद करते हैं।

इंटरैक्टिव PDF फ़ॉर्म इस समस्या को हल करते हैं, जिससे उपयोगकर्ता सीधे फ़ॉर्म फ़ील्ड्स में टाइप कर सकते हैं, आपके दस्तावेज़ अधिक पेशेवर और उपयोगकर्ता‑मित्र बनते हैं। इस व्यापक गाइड में, **आप सीखेंगे कि Java और GroupDocs.Annotation API का उपयोग करके PDF फ़ॉर्म फ़ील्ड्स को कैसे कस्टमाइज़ करें**, ताकि आपके PDFs आपके लिए अधिक काम करें।

**आप अंत तक क्या सीखेंगे:**
- अपने Java प्रोजेक्ट में GroupDocs.Annotation सेटअप करना (यह सोच से आसान है)
- इंटरैक्टिव टेक्स्ट फ़ील्ड्स बनाना जिन्हें उपयोगकर्ता वास्तव में उपयोग कर सकें
- फ़ॉर्म फ़ील्ड्स को आपके ब्रांड और आवश्यकताओं के अनुसार कस्टमाइज़ करना
- डेवलपर्स को अक्सर उलझन में डालने वाले सामान्य मुद्दों का समाधान
- बड़े दस्तावेज़ों के लिए प्रदर्शन अनुकूलन

आइए तुरंत शुरू करते हैं और आपके PDFs को आपके लिए अधिक काम करने लगते हैं।

## त्वरित उत्तर
- **मुख्य लाइब्रेरी कौन सी है?** GroupDocs.Annotation for Java  
- **क्या मैं fillable pdf java बना सकता हूँ?** हाँ – API आपको प्रोग्रामेटिकली fillable फ़ील्ड्स जोड़ने देता है।  
- **क्या मुझे लाइसेंस चाहिए?** विकास के लिए फ्री ट्रायल काम करता है; उत्पादन के लिए एक कमर्शियल लाइसेंस आवश्यक है।  
- **मैं वैधता कैसे जोड़ूँ?** API की `pdf form field validation` सुविधाओं या कस्टम Java लॉजिक का उपयोग करें।  
- **कौन सा Java संस्करण आवश्यक है?** JDK 8+ (JDK 11+ की सिफ़ारिश की जाती है)।

```xml
<repositories>
```

## कार्यान्वयन गाइड: अपना पहला इंटरैक्टिव PDF फ़ॉर्म बनाना

अब मज़े का हिस्सा - वास्तव में इंटरैक्टिव PDF फ़ॉर्म फ़ील्ड्स बनाना जो आपके उपयोगकर्ताओं को पसंद आएँगे। हम प्रत्येक चरण को समझेंगे, केवल "कैसे" नहीं बल्कि प्रत्येक निर्णय के "क्यों" को भी बताएँगे।

### चरण 1: अपनी आउटपुट डायरेक्टरी सेट करें

सबसे पहले - तय करें कि आपका एनोटेटेड PDF कहाँ रहेगा:

```java
String outputPath = YOUR_OUTPUT_DIRECTORY + "/AddTextFieldAnnotation.pdf";
```

**महत्वपूर्ण**: `YOUR_OUTPUT_DIRECTORY` को अपने वास्तविक डायरेक्टरी पाथ से बदलें। एक सामान्य गलती रिलेटिव पाथ्स का उपयोग करना है जो एप्लिकेशन डिप्लॉय करने पर टूट जाते हैं। उत्पादन में पाथ्स के लिए सिस्टम प्रॉपर्टीज़ या एनवायरनमेंट वेरिएबल्स का उपयोग करने पर विचार करें।

### चरण 2: Annotator को इनिशियलाइज़ करें

यहीं से जादू शुरू होता है। `Annotator` क्लास PDF में इंटरैक्टिव एलिमेंट्स जोड़ने के लिए आपका मुख्य टूल है:

```java
final Annotator annotator = new Annotator(YOUR_DOCUMENT_DIRECTORY + "/input.pdf");
```

**यहाँ क्या हो रहा है**: Annotator आपके PDF को मेमोरी में लोड करता है और संशोधन के लिए तैयार करता है। सुनिश्चित करें कि आपका इनपुट PDF मौजूद है और पढ़ा जा सकता है - इस चरण में सबसे आम त्रुटि फ़ाइल न मिलने की अपवाद है।

### चरण 3: कॉन्टेक्स्चुअल रिप्लाईज़ बनाएं (वैकल्पिक लेकिन शक्तिशाली)

रिप्लाईज़ आपके फ़ॉर्म फ़ील्ड्स में संदर्भ और निर्देश जोड़ते हैं। ये जटिल फ़ॉर्म्स के लिए अत्यंत उपयोगी होते हैं:

```java
Reply reply1 = new Reply();
reply1.setComment("First comment");
reply1.setRepliedOn(Calendar.getInstance().getTime());

Reply reply2 = new Reply();
reply2.setComment("Second comment");
reply2.setRepliedOn(Calendar.getInstance().getTime());

List<Reply> replies = new ArrayList<>();
replies.add(reply1);
replies.add(reply2);
```

**रिप्लाईज़ कब उपयोग करें**: इन्हें टूलटिप्स या हेल्प टेक्स्ट के रूप में सोचें। ये भरने के निर्देश, फ़ॉर्मेट आवश्यकताएँ, या अतिरिक्त संदर्भ प्रदान करने के लिए आदर्श हैं जो उपयोगकर्ताओं को आपका फ़ॉर्म सही ढंग से पूरा करने में मदद करता है।

### चरण 4: अपने TextField एनोटेशन को कॉन्फ़िगर करें

यहीं पर आप तय करते हैं कि आपका इंटरैक्टिव फ़ॉर्म फ़ील्ड कैसे दिखेगा और व्यवहार करेगा:

```java
TextFieldAnnotation textField = new TextFieldAnnotation();
textField.setBackgroundColor(65535); // Yellow background color
textField.setBox(new Rectangle(100, 100, 100, 100)); // Position and size
textField.setCreatedOn(Calendar.getInstance().getTime()); // Creation time
textField.setText("Some text"); // Text inside the field
textField.setFontColor(65535); // Yellow font color
textField.setFontSize((double)12); // Font size
textField.setMessage("This is a text field annotation"); // Annotation message
textField.setOpacity(0.7); // Opacity level
textField.setPageNumber(0); // Page number for the annotation
textField.setPenStyle(PenStyle.DOT); // Pen style for border
textField.setPenWidth((byte)3); // Pen width
textField.setReplies(replies); // Attach replies to the annotation
```

**आइए प्रमुख सेटिंग्स को देखें:**
- **स्थिति (`setBox`)**: Rectangle पैरामीटर (x, y, width, height) होते हैं। कोऑर्डिनेट (0,0) आमतौर पर पेज के बॉटम‑लेफ़्ट कोने को दर्शाता है
- **रंग**: RGB वैल्यूज़ या प्री‑डिफाइंड कलर कॉन्स्टैंट्स का उपयोग करें। Yellow (65535) फ़ॉर्म फ़ील्ड्स के लिए अच्छा काम करता है क्योंकि यह दिखने में स्पष्ट है लेकिन चौंकाने वाला नहीं
- **फ़ॉन्ट साइज**: पढ़ने योग्य रखें - 12pt एक अच्छा डिफ़ॉल्ट है, लेकिन अपने दर्शकों और दस्तावेज़ के आकार को ध्यान में रखें
- **अपारदर्शिता**: 0.7 (70%) अच्छी दृश्यता प्रदान करता है बिना मूल सामग्री को अधिक दबाए

### चरण 5: अपने दस्तावेज़ में एनोटेशन जोड़ें

अपने टेक्स्ट फ़ील्ड को कॉन्फ़िगर करने के बाद, इसे PDF में जोड़ें:

```java
annotator.add(textField);
```

यह चरण आपके एनोटेशन को दस्तावेज़ में रजिस्टर करता है। आप विभिन्न एनोटेशन ऑब्जेक्ट्स के साथ `add()` को कई बार कॉल करके कई एनोटेशन जोड़ सकते हैं।

### चरण 6: सहेजें और साफ़‑सफ़ाई करें

अंत में, अपना काम सहेजें और सिस्टम संसाधनों को मुक्त करें:

```java
annotator.save(outputPath);
annotator.dispose();
```

**महत्वपूर्ण**: हमेशा `dispose()` कॉल करें! इसे भूलने से लंबी‑चलाने वाली एप्लिकेशन्स में मेमोरी लीक हो सकता है। यह एक अच्छी प्रैक्टिस है कि try-with-resources या finally ब्लॉक्स का उपयोग करके सुनिश्चित करें कि अपवाद होने पर भी सफ़ाई हो।

## PDF फ़ॉर्म फ़ील्ड्स को कैसे कस्टमाइज़ करें

जब आप **PDF फ़ॉर्म फ़ील्ड्स को कस्टमाइज़** करते हैं, तो आप दृश्य स्टाइलिंग से लेकर इंटरैक्शन व्यवहार तक सब कुछ नियंत्रित कर सकते हैं। ऊपर दिखाए गए रंग, अपारदर्शिता, और पेन स्टाइल सेटिंग्स का उपयोग करके फ़ील्ड्स को अपने ब्रांड के साथ संरेखित करें। आप डिफ़ॉल्ट टेक्स्ट, प्लेसहोल्डर, और रिप्लाईज़ के माध्यम से टूल‑टिप्स भी सेट कर सकते हैं ताकि अंतिम उपयोगकर्ताओं को मार्गदर्शन मिले।

## Java में Fillable PDF कैसे जनरेट करें

कोड स्निपेट्स पहले ही दिखाते हैं कि `TextFieldAnnotation` ऑब्जेक्ट्स जोड़कर **Java में fillable PDF कैसे जनरेट करें**। प्रत्येक आवश्यक फ़ील्ड के लिए `add()` कॉल को दोहराकर, आप पूरी तरह से Java में जटिल, मल्टी‑पेज फ़ॉर्म बना सकते हैं।

## PDF फ़ॉर्म फ़ील्ड वैधता टिप्स

जबकि GroupDocs.Annotation दृश्य एनोटेशन पर केंद्रित है, आप **PDF फ़ॉर्म फ़ील्ड वैधता** को लागू कर सकते हैं:
- उपयोगकर्ता द्वारा पूर्ण PDF जमा करने के बाद सर्वर‑साइड चेक जोड़ना।
- PDF में एम्बेडेड JavaScript (इस ट्यूटोरियल के दायरे से बाहर) का उपयोग करके क्लाइंट‑साइड वैधता।
- दस्तावेज़ सहेजने से पहले इनपुट की लंबाई, फ़ॉर्मेट, या आवश्यक फ़ील्ड्स की वैधता।

## कब TextField एनोटेशन को अन्य विकल्पों के ऊपर चुनें

हर इंटरैक्टिव एलिमेंट को टेक्स्ट फ़ील्ड होना ज़रूरी नहीं है। यहाँ वह समय है जब TextField एनोटेशन आपका सबसे अच्छा विकल्प होते हैं:

**उपयुक्त हैं:**
- नाम और पता फ़ील्ड्स
- टिप्पणी और फीडबैक सेक्शन
- सिंगल‑लाइन डेटा एंट्री
- कस्टमाइज़ेबल यूज़र इनपुट एरिया

**उपयुक्त नहीं हैं:**
- हाँ/ना प्रश्न (इसके बजाय चेकबॉक्स उपयोग करें)
- मल्टीपल चॉइस चयन (रेडियो बटन बेहतर काम करते हैं)
- डेट चयन (डेट पिकर पर विचार करें)
- लंबा टेक्स्ट (टेक्स्ट एरिया अधिक उपयुक्त हैं)

## सामान्य समस्याएँ और ट्रबलशूटिंग

अनुभवी डेवलपर्स भी इन समस्याओं का सामना करते हैं। यहाँ सबसे आम समस्याओं को कैसे हल करें:

### समस्या: एनोटेशन PDF में नहीं दिख रहे हैं

**लक्षण**: आपका कोड बिना त्रुटियों के चलता है, लेकिन PDF अपरिवर्तित दिखता है।

**समाधान:**
1. **पेज नंबर जांचें**: सुनिश्चित करें कि `setPageNumber()` वास्तविक पेज से मेल खाता है (ध्यान रखें, यह ज़ीरो‑इंडेक्स्ड है)
2. **पोजिशनिंग सत्यापित करें**: सुनिश्चित करें कि आपके Rectangle कोऑर्डिनेट्स पेज की सीमाओं के भीतर हैं
3. **फ़ाइल अनुमतियों की पुष्टि करें**: सुनिश्चित करें कि आपका आउटपुट डायरेक्टरी लिखने योग्य है

### समस्या: टेक्स्ट फ़ील्ड बहुत छोटे हैं या गलत स्थान पर हैं

**लक्षण**: फ़ॉर्म फ़ील्ड अनपेक्षित स्थानों पर दिखते हैं या उपयोग करने में कठिन होते हैं।

**समाधान:**
1. **कोऑर्डिनेट सिस्टम समझें**: PDF कोऑर्डिनेट्स अक्सर बॉटम‑लेफ़्ट से शुरू होते हैं, टॉप‑लेफ़्ट से नहीं
2. **विज़िबल बॉर्डर के साथ टेस्ट करें**: अस्थायी रूप से पेन की चौड़ाई बढ़ाएँ और अपारदर्शिता घटाएँ ताकि सटीक पोजिशनिंग देख सकें
3. **टेस्टिंग के लिए PDF व्यूअर्स का उपयोग करें**: विभिन्न PDF व्यूअर्स एनोटेशन को थोड़ा अलग रेंडर कर सकते हैं

### समस्या: बड़े दस्तावेज़ों में मेमोरी समस्याएँ

**लक्षण**: बड़े PDFs में OutOfMemoryError अपवाद या धीमी प्रदर्शन।

**समाधान:**
1. **पेजेज़ को व्यक्तिगत रूप से प्रोसेस करें**: एक बार में पूरे बड़े दस्तावेज़ को लोड न करें
2. **JVM हीप साइज बढ़ाएँ**: अधिक मेमोरी आवंटित करने के लिए `-Xmx` पैरामीटर का उपयोग करें
3. **हमेशा डिस्पोज़ करें**: प्रोसेसिंग के बाद संसाधनों को सही ढंग से रिलीज़ करना सुनिश्चित करें

## प्रदर्शन अनुकूलन टिप्स

उत्पादन में इंटरैक्टिव PDF फ़ॉर्म्स के साथ काम करते समय, प्रदर्शन महत्वपूर्ण है। यहाँ सिद्ध रणनीतियाँ हैं:

### रिसोर्स मैनेजमेंट बेस्ट प्रैक्टिसेज

```java
// Good: Use try-with-resources pattern
try (Annotator annotator = new Annotator(inputPath)) {
    // Your annotation code here
    annotator.save(outputPath);
} // Automatic cleanup
```

### कई एनोटेशन के लिए बैच प्रोसेसिंग

कई Annotator इंस्टेंस बनाने के बजाय, सभी एनोटेशन को एक ही इंस्टेंस में जोड़ें:

```java
Annotator annotator = new Annotator(inputPath);
annotator.add(textField1);
annotator.add(textField2);
annotator.add(textField3);
annotator.save(outputPath);
annotator.dispose();
```

### बड़े दस्तावेज़ों के लिए अनुकूलन

- **प्रति पेज एनोटेशन की सीमा**: प्रति पेज 20‑30 से अधिक फ़ॉर्म फ़ील्ड्स रेंडरिंग को धीमा कर सकते हैं
- **उपयुक्त अपारदर्शिता स्तर उपयोग करें**: कम अपारदर्शिता अधिक प्रोसेसिंग पावर मांगती है
- **पेज‑बाय‑पेज प्रोसेसिंग पर विचार करें**: 100 पेज से अधिक के दस्तावेज़ों को हिस्सों में प्रोसेस करें

## वास्तविक दुनिया के अनुप्रयोग: जहाँ यह वास्तव में उपयोग होता है

इंटरैक्टिव PDF फ़ॉर्म केवल कूल टेक डेमो नहीं हैं - वे वास्तविक व्यापार समस्याओं को हल करते हैं:

### Insurance and Financial Services
ऐसे एप्लिकेशन फ़ॉर्म बनाएं जिन्हें ग्राहक डिजिटल रूप से भर सकें, जिससे प्रोसेसिंग समय दिनों से घंटों में घट जाए। पॉलिसी नंबर, कवरेज राशि, और सिग्नेचर फ़ील्ड्स पूरे वर्कफ़्लो को सुव्यवस्थित करते हैं।

### Human Resources and Onboarding
इंटरैक्टिव फ़ॉर्म्स के साथ नए कर्मचारी कागज़ात आसान हो जाता है। इमरजेंसी कॉन्टैक्ट्स, डायरेक्ट डिपॉज़िट जानकारी, और बेनिफिट चयन सभी डिजिटल रूप से पूरे किए जा सकते हैं।

### Legal Document Processing
कॉन्ट्रैक्ट्स, एग्रीमेंट्स, और लीगल फ़ॉर्म्स को इंटरैक्टिव फ़ील्ड्स से बहुत लाभ मिलता है। क्लाइंट्स डेट्स, सिग्नेचर, और विशिष्ट शर्तें बिना लीगल सॉफ़्टवेयर के भर सकते हैं।

### Educational Materials and Assessments
इंटरैक्टिव वर्कशीट्स, एप्लिकेशन फ़ॉर्म्स, और असेसमेंट दस्तावेज़ बनाएं जिन्हें छात्र डिजिटल रूप से पूरा कर सकें, जिससे ग्रेडिंग और फीडबैक अधिक प्रभावी हो जाता है।

### Healthcare and Patient Forms
पेशेंट इंटेक फ़ॉर्म्स, मेडिकल हिस्ट्री प्रश्नावली, और कंसेंट फ़ॉर्म्स इंटरैक्टिव होने पर अधिक सुलभ और प्रोसेस करने में आसान हो जाते हैं।

## उन्नत कस्टमाइज़ेशन विकल्प

बेसिक को मास्टर करने के बाद, ये उन्नत तकनीकें आपके फ़ॉर्म्स को अगले स्तर पर ले जा सकती हैं:

### ब्रांड कंसिस्टेंसी के लिए कस्टम स्टाइलिंग

अपने फ़ॉर्म फ़ील्ड्स को अपने ब्रांड के रंगों और फ़ॉन्ट्स से मिलाएँ:

```java
textField.setBackgroundColor(0x0066CC); // Brand blue
textField.setFontColor(0xFFFFFF); // White text
textField.setFontSize(14.0); // Larger, more readable text
```

### डायनामिक फ़ील्ड बिहेवियर

ऐसे फ़ील्ड्स कॉन्फ़िगर करें जो उपयोगकर्ता इनपुट पर प्रतिक्रिया दें:

```java
textField.setText("Enter your name here..."); // Placeholder text
textField.setOpacity(0.8); // Slightly more prominent
textField.setPenStyle(PenStyle.SOLID); // Clean, professional border
```

### वैधता और एरर हैंडलिंग

जबकि GroupDocs.Annotation डिस्प्ले को संभालता है, अंतिम PDF में बेहतर यूज़र एक्सपीरियंस के लिए JavaScript वैधता जोड़ने पर विचार करें।

## निष्कर्ष: बेहतर PDF फ़ॉर्म्स की आपकी राह

अब आपके पास Java के साथ इंटरैक्टिव PDF फ़ॉर्म्स बनाने के लिए पूरा टूलकिट है। बेसिक टेक्स्ट फ़ील्ड्स से लेकर उन्नत कस्टमाइज़ेशन तक, आप न केवल इन फीचर्स को लागू करना जानते हैं, बल्कि कब और क्यों उपयोग करना है, यह भी समझते हैं।

**मुख्य बिंदु:**
- इंटरैक्टिव PDF फ़ॉर्म्स उपयोगकर्ता अनुभव को काफी बेहतर बनाते हैं
- उचित सेटअप के साथ GroupDocs.Annotation कार्यान्वयन को सरल बनाता है
- उत्पादन एप्लिकेशन्स के लिए प्रदर्शन अनुकूलन और रिसोर्स मैनेजमेंट महत्वपूर्ण हैं
- वास्तविक दुनिया के अनुप्रयोग स्वास्थ्य देखभाल से वित्त तक विभिन्न उद्योगों में फैले हैं

क्या आप इसे अपने प्रोजेक्ट में लागू करने के लिए तैयार हैं? एक सरल फ़ॉर्म फ़ील्ड से शुरू करें, पूरी तरह टेस्ट करें, और API में अधिक सहज होते ही धीरे‑धीरे जटिलता जोड़ें।

## अक्सर पूछे जाने वाले प्रश्न

**प्रश्न: क्या मैं मौजूदा PDFs में इंटरैक्टिव फ़ॉर्म फ़ील्ड्स जोड़ सकता हूँ?**  
**उत्तर:** बिल्कुल! GroupDocs.Annotation API मौजूदा PDF दस्तावेज़ों के साथ काम करता है। बस `Annotator` क्लास से अपना PDF लोड करें और अपने इंटरैक्टिव फ़ील्ड्स जोड़ें।

**प्रश्न: एक ही PDF में मैं कितने फ़ॉर्म फ़ील्ड्स जोड़ सकता हूँ?**  
**उत्तर:** कोई कठोर सीमा नहीं है, लेकिन प्रदर्शन कारणों से प्रति पेज 50 फ़ील्ड्स से कम रखने पर विचार करें। बहुत अधिक एनोटेशन कुछ व्यूअर्स में PDF रेंडरिंग को धीमा कर सकते हैं।

**प्रश्न: क्या इंटरैक्टिव PDF फ़ॉर्म्स सभी PDF व्यूअर्स में काम करते हैं?**  
**उत्तर:** अधिकांश आधुनिक PDF व्यूअर्स, जैसे Adobe Acrobat, Foxit Reader, और अधिकांश वेब ब्राउज़र्स, इंटरैक्टिव फ़ॉर्म फ़ील्ड्स को सपोर्ट करते हैं। हालांकि, हमेशा अपने लक्ष्य दर्शकों के पसंदीदा व्यूअर्स में टेस्ट करें।

**प्रश्न: क्या मैं फ़ॉर्म फ़ील्ड्स को अपने ब्रांड रंगों के अनुसार स्टाइल कर सकता हूँ?**  
**उत्तर:** हाँ! आप बैकग्राउंड रंग, फ़ॉन्ट रंग, बॉर्डर स्टाइल, और अपारदर्शिता को कस्टमाइज़ करके अपने ब्रांड गाइडलाइन्स के अनुसार बना सकते हैं।

**प्रश्न: TextField एनोटेशन और वास्तविक PDF फ़ॉर्म फ़ील्ड्स में क्या अंतर है?**  
**उत्तर:** TextField एनोटेशन विज़ुअल ओवरले होते हैं जिन्हें भरा जा सकता है, जबकि पारंपरिक PDF फ़ॉर्म फ़ील्ड्स दस्तावेज़ संरचना में एम्बेडेड होते हैं। एनोटेशन अक्सर लागू करने में आसान और कस्टम स्टाइलिंग के लिए अधिक लचीले होते हैं।

**प्रश्न: मैं फ़ॉर्म वैधता और डेटा संग्रह कैसे संभालूँ?**  
**उत्तर:** GroupDocs.Annotation विज़ुअल प्रस्तुति को संभालता है। वैधता और डेटा संग्रह के लिए, आप आमतौर पर सर्वर‑साइड पर एनोटेशन डेटा निकालते हैं या PDF के भीतर JavaScript का उपयोग करते हैं।

**प्रश्न: क्या मैं कनेक्टेड फ़ील्ड्स के साथ मल्टी‑पेज फ़ॉर्म बना सकता हूँ?**  
**उत्तर:** हाँ, आप कई पेजों में एनोटेशन जोड़ सकते हैं। प्रत्येक एनोटेशन अपना पेज नंबर निर्दिष्ट करता है, इसलिए आप व्यापक मल्टी‑पेज फ़ॉर्म बना सकते हैं।

**प्रश्न: PDF के अलावा कौन से फ़ाइल फ़ॉर्मेट इंटरैक्टिव एनोटेशन को सपोर्ट करते हैं?**  
**उत्तर:** GroupDocs.Annotation विभिन्न फ़ॉर्मेट्स जैसे Word दस्तावेज़, Excel स्प्रेडशीट, और इमेज फ़ाइल्स को सपोर्ट करता है, हालांकि इंटरैक्टिव फ़ॉर्म्स के लिए PDF सबसे आम है।
