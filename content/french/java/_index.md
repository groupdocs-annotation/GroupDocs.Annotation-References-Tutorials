## Aperçu des capacités principales

### Comment **annotate PDF Java** des documents avec GroupDocs.Annotation
L'API propose un flux de travail simple : charger le document, créer un objet d'annotation, configurer son apparence, puis enregistrer le résultat. Vous trouverez ci‑dessous une description de haut niveau de chaque étape (des exemples de code sont disponibles dans les tutoriels liés).

### Annotation d'image Java – Ajouter des graphiques à vos documents
Vous pouvez intégrer des images raster ou vectorielles en tant qu'annotations, les positionner avec ressources commentaires visuels.

### Ajouter des annotations Java – Marquage texte et graphique de base
Des simples surlignages aux formes complexes, la bibliothèque propose un modèle unifié pour tous les types d'annotation, facilitant le passage du texte au marquage graphique dans le même document.

### Charger des documents Java – Ingestion de documents efficace
GroupDocs.Annotation prend en charge le chargement depuis des fichiers locaux, des flux, le stockage cloud (Amazon S3, Azure Blob), des URL et des serveurs FTP. Cette flexibilité garantit que vous pouvez **load documents java** depuis n'importe quelle source utilisée par votre application.

### Gestion des annotations Java – Contrôler le cycle de vie des annotations
Créez, mettez à jour, supprimez et filtrez les annotations par programme. Vous pouvez également joindre des métadonnées personnalisées, définir les informations de l'auteur et appliquer des politiques de sécurité.
