## คู่มือการใช้งาน

### การเพิ่มคำอธิบายพื้นที่

คำอธิบายพื้นที่ช่วยให้คุณสามารถเน้นส่วนต่างๆ ของเอกสารได้ มาดูวิธีนำฟีเจอร์นี้ไปใช้กัน

#### ภาพรวม

คำอธิบายพื้นที่เหมาะอย่างยิ่งสำหรับการทำเครื่องหมายพื้นที่สี่เหลี่ยมใน PDF โดยมักใช้ในการวิจารณ์หรือเมื่อชี้ให้เห็นเนื้อหาที่เจาะจง

#### การดำเนินการแบบทีละขั้นตอน

**1. กำหนดคำอธิบายประกอบ**

ประการแรก ให้สร้างอินสแตนซ์ของ `AreaAnnotation`ซึ่งเกี่ยวข้องกับการระบุพิกัดและขนาดของพื้นที่ที่คุณต้องการใส่คำอธิบายประกอบ

```csharp
using GroupDocs.Annotation.Models;
using GroupDocs.Annotation.Models.AnnotationModels;

// สร้างคำอธิบายพื้นที่ใหม่
AreaAnnotation area = new AreaAnnotation {
    Box = new Rectangle(100, 100, 200, 100), // X, Y, ความกว้าง, ความสูง
    BackgroundColor = 65535, // สีเหลืองในรูปแบบ ARGB
    PageNumber = 0, // หน้าแรก (ดัชนีฐานศูนย์)
    CreatedOn = DateTime.Now,
    Message = "This area requires review",
};
```

**2. เพิ่มคำอธิบายลงในเอกสาร**

ขั้นตอนต่อไป เพิ่มคำอธิบายประกอบนี้ลงในเอกสารของคุณโดยใช้ `Annotator` วัตถุ.

```csharp
using (Annotator annotator = new Annotator(inputPath)) {
    annotator.Add(area);
    annotator.Save(outputPath); // บันทึกโดยใช้คำอธิบายประกอบ
}
```

**3. คำอธิบายพารามิเตอร์**

- **กล่อง**: กำหนดตำแหน่งและขนาดของพื้นที่
- **สีพื้นหลัง**:กำหนดสีคำอธิบายประกอบ ใช้รูปแบบ ARGB เพื่อความแม่นยำ
- **หมายเลขหน้า**: ระบุว่าต้องการใส่คำอธิบายหน้าใด (ดัชนีฐานศูนย์)
- **สร้างเมื่อ**:วันที่และเวลาที่สร้างคำอธิบายประกอบ

#### เคล็ดลับการแก้ไขปัญหา

- **ปัญหาเรื่องสี**: ตรวจสอบให้แน่ใจว่าคุณใช้ค่า ARGB ที่ถูกต้อง
- **ปัญหาการวางตำแหน่ง**: ตรวจสอบว่าพิกัดของคุณตรงกับขนาดของเอกสาร

## การประยุกต์ใช้งานจริง

GroupDocs.Annotation สามารถรวมเข้ากับเวิร์กโฟลว์ต่างๆ ได้:

1. **ระบบตรวจสอบเอกสาร**:สร้างคำอธิบายประกอบอัตโนมัติระหว่างการตรวจสอบโดยเพื่อนร่วมงาน
2. **เครื่องมือทางการศึกษา**:เน้นส่วนที่สำคัญในสื่อการเรียนรู้
3. **เอกสารทางกฎหมาย**:ทำเครื่องหมายจุดสำคัญสำหรับการตรวจสอบทางกฎหมาย
4. **การพัฒนาซอฟต์แวร์**:ใส่คำอธิบายใน PDF ด้วยข้อกำหนดการออกแบบหรือตัวอย่างโค้ด
